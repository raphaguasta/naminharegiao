Crie um commit semântico para as mudanças atuais.

1. Rode `cd app && npm run build` — corrija qualquer erro de tipo antes de continuar.
2. Analise o diff (`git diff` e `git status`).
3. Escreva a mensagem em português, formato: `tipo(escopo): descrição`
   - Tipos: feat, fix, refactor, style, chore
   - Escopos: portal, admin, data, routing, ui, seo
4. Se houver mudanças em contextos muito diferentes, proponha commits separados.
5. Execute o commit.
